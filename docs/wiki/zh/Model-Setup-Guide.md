# 模型接入指南

这页说明如何用比较稳的方式把 AI 模型接入 GEOFlow。

## 1. 模型接入入口

后台路径：

`AI 配置中心 -> AI 模型管理`

这里负责：

- 新增模型
- 修改模型
- 设置模型类型
- 配置 API 地址与模型 ID
- 设置启用状态
- 配置智能模型切换优先级

## 2. 需要填写什么

通常至少要填写：

- 模型名称
- API 地址
- 模型 ID
- Bearer Token / API Key
- 模型类型

当前最常用的是 `chat` 模型。

## 3. API 地址规则

GEOFlow 支持两种写法：

- 服务商基础地址
- 完整接口 URL

系统会按模型能力自动补路径：

- chat 默认补全 `/v1/chat/completions`
- embedding 默认补全 `/v1/embeddings`

对于版本化基础路径，系统也做了兼容，例如：

- 智谱 `/api/paas/v4`
- 火山方舟 `/api/v3`

## 4. 模型选择建议

上手阶段建议：

- 先接一个稳定的 chat 模型
- 先跑通标题生成和正文生成
- 再考虑更复杂的 provider 组合

不要在第一轮就追求：

- 很多模型同时接入
- 很复杂的 provider 组合
- 很激进的自动切换策略

## 5. 深度思考模型注意事项

部分 reasoning / 深度思考模型在响应前会静默较长时间。  
因此要注意：

- AI 请求超时不宜过短
- 不要使用过于激进的低速熔断
- 标题生成和正文生成最好都按统一请求策略处理

如果模型很强但响应慢，不一定适合用在所有场景。  
例如标题生成通常更适合快模型，而不是最重的推理模型。

## 6. 智能模型切换

GEOFlow 现在支持两种任务模式：

- `fixed`
- `smart_failover`

其中：

- `fixed`：始终使用指定主模型
- `smart_failover`：主模型失败后按优先级尝试下一个可用聊天模型

建议：

- 先把主模型跑稳
- 再启用智能切换
- 把优先级设置清楚

## 7. 接入后的最小验证

建议至少验证这些：

1. 模型保存成功
2. 标题生成成功
3. 正文生成成功
4. 任务能正常使用该模型
5. 如启用智能切换，主模型失败时能切后备模型

## 8. 常见问题

### 模型 404

常见原因：

- 填了错误的 base URL
- provider 实际不是 `/v1` 风格路径
- 模型 ID 填写错误

### 模型超时

常见原因：

- reasoning 模型响应慢
- 请求超时策略过短
- provider 网络波动

### 模型能保存但任务失败

常见原因：

- 模型类型不匹配
- Token 无效
- 额度限制
- provider 返回结构异常

## 9. 推荐原则

一句话：

> 先接稳定模型，先验证链路，再做复杂编排。

模型接入是系统能力的一部分，不是越多越好，而是越稳定越有价值。
