# 部署检查清单

这页不是完整部署文档，而是一份上线前和上线后的核对清单。

## 一、基础环境

确认这些前提已经满足：

- PHP 版本符合当前公开仓要求
- PostgreSQL 已可正常连接
- `curl`、`pdo_pgsql` 等核心扩展已启用
- Web、Scheduler、Worker 运行链路已经打通
- 服务器时区已确认

如果是 Docker 部署，还要确认：

- `docker compose` 能正常运行
- 容器之间网络互通
- `web / postgres / scheduler / worker` 都已启动

## 二、环境变量与安全项

上线前至少检查这些：

- `APP_SECRET_KEY` 已改成随机长字符串
- 默认管理员密码已修改
- `SITE_URL` 与真实访问域名一致
- 端口、反向代理和 HTTPS 配置正确
- 不在仓库或公开页面里暴露真实 API Key

## 三、数据库与运行状态

确认：

- 数据库表已初始化完成
- 后台可以正常登录
- 模型、任务、素材等页面可以正常读写
- Scheduler 能正常入队
- Worker 能正常执行
- 日志中没有持续报错

## 四、AI 模型与生成链路

上线前建议至少验证一条完整链路：

1. 模型配置可保存
2. 标题生成可跑通
3. 正文生成可跑通
4. 文章能进入草稿
5. 审核 / 发布链路正常

如果你启用了深度思考模型或智能模型切换，还应额外确认：

- AI 请求超时策略与模型响应时间匹配
- 主模型失败时，后备模型切换能正常工作

## 五、前台输出

至少检查这几类页面：

- 首页
- 分类页
- 文章详情页
- 归档页

同时确认：

- `title / description / keywords` 正常
- Open Graph 正常
- 结构化数据正常
- 文章图片和列表摘要显示正常
- 前台模板和主题切换没有破坏数据契约

## 六、内容质量与知识库

在业务层面，建议上线前确认：

- 知识库内容真实、可验证
- 提示词没有明显鼓励虚假生成
- 任务目标清晰，不是无节制堆量
- 输出内容有审核或抽查机制

这里的底线不是“能生成就行”，而是“输出要值得被用户和 AI 信任”。

## 七、监控与回滚准备

上线前建议准备：

- 日志查看方式
- 任务失败排查路径
- 如何暂停任务
- 如何关闭某个模型
- 如何切回默认前台模板

如果你启用了主题包：

- 保留默认前台
- 保留 preview 路由
- 确认后台可一键切回默认模板

## 八、推荐上线顺序

最稳的顺序是：

1. 先在本地或测试环境跑通
2. 再做一轮小规模真实内容验证
3. 再上线正式域名
4. 再逐步扩大任务量和自动化范围

不要一开始就：

- 高并发跑很多任务
- 上很多模型
- 同时切多个模板
- 批量生成大量未审核内容

先确保系统稳定，再追求规模。
