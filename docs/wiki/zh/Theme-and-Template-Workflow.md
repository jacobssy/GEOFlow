# 模板与主题工作流

GEOFlow 的模板能力不是“直接替换前台代码”，而是遵循 preview-first 的工作流。

## 1. 主题包的边界

主题包只负责前台展示层，包括：

- 页头
- 页尾
- 首页
- 分类页
- 文章页
- 归档页
- 主题样式与视觉 token

它不应该破坏这些系统契约：

- 路由
- 分类 / 文章 / 归档查询逻辑
- SEO 规则
- Open Graph
- 结构化数据
- 多语言逻辑

## 2. 主题包结构

当前推荐的主题包结构是：

```text
themes/<theme-id>/
├── manifest.json
├── tokens.json
├── mapping.json
├── assets/
│   └── theme.css
└── templates/
    ├── header.php
    ├── footer.php
    ├── home.php
    ├── category.php
    ├── article.php
    └── archive.php
```

其中：

- `manifest.json`：主题元信息
- `tokens.json`：颜色、圆角、阴影、字体等设计 token
- `mapping.json`：模块映射关系
- `theme.css`：主题样式

## 3. 推荐工作流

最稳的模板工作流是：

1. 先确定参考站点或目标风格
2. 用 `geoflow-template` 做模块映射和 token 提取
3. 先生成主题包
4. 先走动态预览
5. 预览确认后，再到后台启用

这套流程的意义是：

- 不直接破坏线上前台
- 能基于真实数据预览
- 能多轮微调
- 能保留回滚路径

## 4. 动态预览路由

当前支持：

- `/preview/<theme-id>/`
- `/preview/<theme-id>/category/<slug>`
- `/preview/<theme-id>/article/<slug>`
- `/preview/<theme-id>/archive`

预览使用的是真实数据库数据，不是静态示例页。

## 5. 后台启用逻辑

在后台 `网站设置` 中：

- 可以查看当前启用模板
- 可以预览首页 / 分类页 / 文章页 / 归档页
- 可以切回默认前台
- 可以启用某个主题包

建议：

- 预览通过后再启用
- 启用前保留默认模板
- 出现问题时优先回切默认模板

## 6. 列表摘要和 Markdown 清洗

模板层已经支持对首页、分类页、归档页的列表摘要做展示清洗：

- 去掉 `#`
- 去掉 `**`
- 去掉列表符号和链接残留

这样列表卡片显示的是可阅读摘要，而不是原始 Markdown 符号。

## 7. 什么时候适合做模板化

适合在这些阶段进行：

- 内容链路已经跑通
- 知识库和任务结构已经稳定
- 前台已经有明确的风格目标
- 需要多站点 / 多频道 / 多主题输出

如果内容本身还不稳定，先不要把大量精力投到主题层。

## 8. 推荐原则

一句话：

> 模板应该服务于内容系统，而不是反过来绑架内容系统。

所以：

- 先系统稳定
- 再模板化
- 先预览
- 再启用
